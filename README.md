## üìò Overview
The **Medical Inventory Management System** is a Salesforce-based application designed to streamline hospital and clinic inventory operations.  
It automates the tracking, request, and approval of medical supplies, ensuring real-time visibility, accuracy, and stock availability across departments.

This project leverages **Salesforce Custom Objects, Apex Triggers, Flows, Lightning Components, and Reports** to enable complete control over medical inventory lifecycles.

---

## üéØ Objectives
- Maintain real-time data of all medical items and quantities.  
- Reduce stock wastage through automatic expiry and low-stock alerts.  
- Enable smooth item request and approval workflows.  
- Provide dashboards for data-driven management decisions.  
- Improve efficiency and accuracy in inventory handling.

## üß† Functional Workflow
1. **Department User** creates a new request for an inventory item.  
2. **System** validates the request based on available quantity.  
3. **Inventory Manager** approves or rejects the request.  
4. Upon approval, **Apex Trigger** updates stock levels automatically.  
5. **Notifications** are sent to relevant users.  
6. **Dashboards** update in real-time to reflect stock and usage data.
